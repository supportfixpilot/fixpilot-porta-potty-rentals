# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Static SEO site for **FixPilot Porta Potty Rentals** (fixpilotportapottyrentals.com), deployed to Cloudflare Pages. Plain HTML + Tailwind via CDN. No build step, no test suite, no package.json. Python scripts at the root are one-off content tooling, not part of a runtime.

The whole site exists to rank locally — ~200 city landing pages, ~20 service pages, and a blog all funnel to phone calls.

## Commands

```bash
# Serve locally with clean URL support (mirrors Cloudflare Pages behavior)
python3 serve.py           # http://localhost:8000
python3 serve.py 3000      # custom port

# NOTE: Do NOT use 'python3 -m http.server' — it cannot resolve clean URLs
# like /locations, /blog/post-name, or /services/page-name (returns 404).

# Rebuild Tailwind CSS after any HTML changes
npm run build   # or: node_modules/.bin/tailwindcss -i src/tw-input.css -o assets/tw.css --minify

# Create a stub city folder (legacy — see "Page layout" caveat below)
python3 script.py "City Name"

# Populate hero + service images in a city page
python3 populate_images.py <city-folder> [--hero-dir ./hero-banner-images] [--service-dir ./service-images]

# Ping IndexNow (Bing/Yandex/Seznam). Set INDEXNOW_KEY env var first;
# place <KEY>.txt at the site root. Args optional; defaults to whole sitemap.
python3 indexnow_ping.py [URL ...]

# Regenerate a batch of city pages from the Atlanta template (edit CITY_DATA in the script first)
python3 regenerate_pages.py
```

`.venv/` and `venv/` both exist — `.venv/` is the active one.

## Page layout (important — AGENTS.md is stale on this)

`AGENTS.md` describes `locations/{city}/index.html`. **That is not the current layout.** City pages now live at the **repo root** as:

```
porta-potty-rental-{city-slug}-{state}/index.html
```

The `.htaccess` rewrite (`/porta-potty-rental-{city}-{state}/` → `/locations/{city}/index.html`) is leftover from the old layout and does not match current files. On Cloudflare Pages the URL `/porta-potty-rental-austin-tx` serves `porta-potty-rental-austin-tx/index.html` directly via folder-index resolution; the `.htaccess` is only meaningful on an Apache host. Treat `.htaccess` as historical unless someone explicitly switches hosting.

Image paths inside city pages are `../hero-banner-images/...` and `../service-images/{service-slug}/...` — one level up because the city folder is at the repo root.

If a script (e.g. `script.py`, `populate_images.py`) still writes to `locations/`, it's targeting the old layout. New city pages should be created at the root following the `porta-potty-rental-{slug}-{state}/` pattern. Several helper scripts (`restore_and_fix.py`, `fix_service_links.py`) hardcode `BASE = "/Users/hasanulrubel/..."` — a different user's path. Those are stale one-offs, not reusable utilities.

## Site structure

- `index.html` (root) — main landing page, ~390KB. Edit cautiously; it's the SEO crown jewel.
- `locations.html` — directory page linking all city pages.
- `porta-potty-rental-{city}-{state}/` — 200+ city landing pages, each self-contained with its own schema.org LocalBusiness JSON-LD, geo meta, OG tags, canonical URL.
- `services/{service-slug}.html` — 19 service-specific pages (standard, deluxe, luxury trailers, ADA, hand wash, construction, crane-hook, septic, etc.). Canonical URLs follow `/services/{slug}`.
- `blog/` — long-form SEO content (cost guides, permits, calculators).
- `hero-banner-images/` — pool of hero images; `populate_images.py` picks one at random per city page.
- `service-images/{service-slug}/` — per-service image pools; matched to city-page service cards by alt-text → folder name fuzzy match.
- `sitemap.xml`, `robots.txt` — regenerated by `python3 sitemap_gen.py` from on-disk files (per-page lastmod from file mtime). Run before each deploy.

## City page conventions

Every city page has, and must keep in sync:
- `<title>`, `<meta name="description">`, `<link rel="canonical">` referencing `/porta-potty-rental-{slug}-{state}`
- OG + Twitter card tags pointing at a hero image URL on the canonical domain
- `geo.region`, `geo.placename`, `geo.position`, `ICBM` meta
- A `<section id="services">` containing service cards. `link_service_cards.py` maps each card's `<h3>` text to a `/services/{slug}` URL using keyword rules — when adding a new service variant, extend `map_service_name_to_url` there.
- schema.org LocalBusiness JSON-LD with `addressLocality`, `addressRegion`, `postalCode`, `hasMap` — `populate_images.py update_google_maps` rewrites the embedded Google Maps iframe from these fields.

## Content-generation flow

1. **New city**: create `porta-potty-rental-{slug}-{state}/index.html` at the root (typically by copying a nearby existing city, e.g. Atlanta or Austin, and editing).
2. **Bulk regen**: extend `CITY_DATA` in `regenerate_pages.py` and run it — it stamps the Atlanta template with city-specific name/state/county/areas/theme color.
3. **Images**: `populate_images.py` randomizes the hero, rewrites the maps iframe, and matches each service card's `alt` text to a `service-images/` subfolder.
4. **Service-card links**: `link_service_cards.py` wraps each service card's CTA in a link to the matching `/services/{slug}` page.
5. **Sitemap**: regenerate via `python3 sitemap_gen.py` after content changes.
6. **Submit**: use Google Search Console for the canonical submission. For Bing/Yandex use `python3 indexnow_ping.py` (requires `INDEXNOW_KEY` env var and a `<KEY>.txt` file at site root).

## Quirks to know about

- `__pycache__/` is checked in (alongside compiled .pyc for `link_service_cards`). Safe to delete; will regenerate.
- The repo has both `venv/` and `.venv/`. New work should use `.venv/`.
- `AGENTS.md` and `GEMINI.md` predate the root-level city-folder layout; trust the actual file tree over either.
